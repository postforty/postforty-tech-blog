# AWS 프리 티어 해지 및 계정 영구 폐쇄 가이드

## 1\. 서론: 프리 티어의 구조적 이해

AWS 프리 티어(Free Tier)는 넷플릭스와 같은 구독형 서비스가 아니기에 별도의 '구독 취소' 버튼이 존재하지 않는다. 이는 신규 가입 계정에 12개월간 제공되는 할인 혜택일 뿐이다. 따라서 사용자가 더 이상 AWS를 이용하지 않고 비용 청구를 원천 차단하려면, 실행 중인 모든 리소스를 종료하거나 계정 자체를 해지(Close)해야 한다. 본 글에서는 과금 폭탄을 방지하고 안전하게 계정을 폐쇄하는 절차를 기술한다.

## 2\. 사전 준비: 리소스 점검 및 데이터 백업

계정 해지를 진행하기 전, 가장 중요한 단계는 숨겨진 리소스를 찾아내는 것이다. 계정을 해지하더라도 미납 요금은 청구될 수 있으며, 데이터는 복구할 수 없게 된다.

### 2-1. 핵심 과금 리소스 확인

가장 빈번하게 비용을 유발하는 5대 리소스를 우선적으로 확인해야 한다. 대시보드에서 '모든 리전(All Regions)'을 대상으로 아래 항목을 점검한다.

- **EC2 (가상 서버)**: 인스턴스 상태가 'Terminated'인지 확인한다. 'Stopped' 상태는 스토리지 비용이 발생한다.
- **RDS (데이터베이스)**: 데이터베이스 인스턴스를 삭제하고, 스냅샷까지 제거해야 한다.
- **S3 (스토리지)**: 버킷을 비우고 버킷 자체를 삭제한다.
- **Elastic IP (탄력적 IP)**: 인스턴스와 연결되지 않은 고정 IP는 시간당 요금이 부과되므로 반드시 릴리스(Release)한다.
- **NAT Gateway**: VPC 설정 시 생성된 게이트웨이는 시간당 고정 비용이 발생하므로 삭제한다.

### 2-2. AWS CLI를 활용한 리소스 조회

콘솔에서 확인이 어렵다면 AWS CLI(Command Line Interface)를 통해 활성 리소스를 조회할 수 있다.

- EC2 인스턴스 조회 명령어
  ```bash
  aws ec2 describe-instances \
      --query "Reservations[*].Instances[*].{ID:InstanceId,State:State.Name}" \
      --output table
  ```
- S3 버킷 리스트 조회 명령어
  ```bash
  aws s3 ls
  ```

### 2-3. 태그 편집기를 이용한 전수 검사

특정 리전에 숨겨진 리소스를 찾기 위해 '리소스 그룹 및 태그 편집기'를 활용하는 것이 효율적이다.

- AWS 콘솔 상단 검색창에 `Resource Groups & Tag Editor`를 입력하여 이동한다.
- 리전 설정을 'All regions'로 변경하고 리소스 유형을 'All supported resource types'로 설정하여 검색한다.
- 검색 결과에서 비용을 유발하지 않는 `DHCPOptions`, `VirtualMFADevice` 등은 무시해도 무방하다.

## 3\. 결제 수단(카드) 삭제 문제 해결

AWS 정책상 기본 결제 수단은 삭제가 불가능하도록 설정되어 있다. 따라서 이 단계는 스킵해도 된다.

### 3-1. 카드 삭제가 불가능한 이유

AWS는 후불제 과금 시스템을 채택하고 있다. 미납 발생을 방지하기 위해 활성 계정에는 최소 1개 이상의 유효한 결제 수단이 등록되어 있어야 한다. 따라서 유일하게 등록된 카드는 삭제 버튼이 비활성화된다.

### 3-2. 카드 정보 삭제를 위한 우회 방법

개인정보 보호를 위해 반드시 카드를 삭제하고 싶다면 '교체 후 삭제' 방식을 사용한다.

- **임시 카드 등록**: 잔액이 없거나 사용하지 않는 체크카드를 새 결제 수단으로 추가한다.
- **기본값 변경**: 새로 등록한 카드를 '기본(Default)'으로 설정한다.
- **기존 카드 삭제**: 이제 주 사용 카드는 기본 수단이 아니므로 삭제가 가능해진다.

## 4\. 계정 해지 실행 및 사후 확인

모든 정리가 끝났다면 실제 계정 폐쇄를 진행한다. 이 작업은 루트 사용자(이메일 로그인) 권한이 필요하다.

### 4-1. 계정 해지 절차

- 우측 상단 계정명을 클릭하고 **[계정(Account)]** 메뉴로 진입한다.
- 페이지 최하단으로 스크롤 하여 **[계정 해지(Close Account)]** 섹션을 찾는다.
- 모든 약관 동의 체크박스를 선택한 후 **[계정 해지]** 버튼을 클릭한다.

### 4-2. 해지 후 상태 확인

해지 버튼을 누른 직후에는 로그인이 해제되거나 '계정 등록 미완료' 화면으로 리다이렉트 될 수 있다. 정확한 해지 여부는 다음 두 가지 방법으로 확인한다.

- **이메일 확인**: AWS로부터 `AWS Account Closed`라는 제목의 확인 메일이 도착했는지 확인한다.
- **콘솔 접속**: 재로그인 시도 시 계정 상태가 `Suspended` 또는 `Closed`로 표시되는지 확인한다. 이때 **[계정 재활성화]** 버튼을 누르지 않도록 주의한다.

#### 4-2-1. 90일 유예 기간

- 계정을 해지하더라도 즉시 데이터가 삭제되는 것은 아니다.
- 90일간은 '해지 유예' 기간으로 설정되며, 이 기간 내에 로그인하여 재활성화하면 계정을 복구할 수 있다.
- 영구적인 데이터 삭제는 90일 이후에 실행된다.
