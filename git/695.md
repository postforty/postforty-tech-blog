## .gitignore가 작동하지 않는다?

**실행 환경**
- OS: Windows 11 / macOS 14
- Git: 2.x 버전 이상

프로젝트를 진행하다 보면 개인 설정 파일이나 IDE 설정 폴더(예: `.vscode/`, `.idea/`)가 원격 저장소에 잘못 올라가는 경우가 있다. 이를 방지하기 위해 뒤늦게 `.gitignore` 파일에 해당 폴더나 파일을 추가하게 된다.

그런데 분명히 `.gitignore`에 추가하고 저장했는데도 불구하고, 로컬에서 파일 내용이 변경될 때마다 `git status`에 수정된 파일(Modified)로 잡히고 계속해서 Git의 추적을 받는 현상이 발생한다.

## 원인 파악: 왜 이런 일이 발생할까?

결론부터 말하면, **Git은 이미 한 번 커밋되어 "추적 중(Tracked)"인 파일에 대해서는 `.gitignore` 규칙을 무시하기 때문**이다.

`.gitignore`의 주된 역할은 **"아직 Git에 추적되지 않은(Untracked) 파일들을 새로 추가하지 못하게 막는 것"**이다. 따라서 과거에 이미 `git add`와 `git commit`을 통해 Git의 관리 대상(Index)에 포함되어버린 파일이라면, 나중에 `.gitignore`에 이름을 적어 넣더라도 Git은 계속 관리해야 하는 파일로 인식하고 변경 사항을 추적하게 된다.

## 해결 방법: Git 캐시(Index) 비우기

이 문제를 해결하려면 실제 로컬 컴퓨터에 있는 파일은 그대로 둔 채, **Git의 추적 내역(캐시)에서만 해당 파일을 삭제**해야 한다.

이를 위해 `git rm --cached` 명령어를 사용한다.

### 1단계: Git 추적 목록에서 삭제하기

터미널을 열고, 프로젝트 폴더 경로에서 아래의 명령어를 입력한다. 다음은 `.vscode` 폴더를 지우는 예시이다.

```bash
# 특정 폴더 전체의 추적을 중지할 때 (-r 옵션은 하위 폴더/파일까지 포함)
git rm -r --cached .vscode

# 단일 파일만 추적을 중지할 때
git rm --cached .env
```

명령어를 실행하면 로컬 작업 디렉토리의 `.vscode` 폴더나 파일은 안전하게 그대로 남아있고, 오직 Git의 추적 리스트에서만 빠지게 된다.

> **주의:** `--cached` 옵션을 빼고 `git rm`만 입력하면 **로컬의 실제 파일까지 모두 삭제되므로 주의해야 한다.**

### 2단계: 변경 사항 커밋하기

이제 `git status`를 입력해 보면, 해당 파일이나 폴더가 삭제(Deleted)된 것으로 표시될 것이다. 
이는 실제 파일이 지워진 것이 아니라 "Git의 관리 대상에서 제외되었다"는 의미이다.

이제 이 상태를 저장소에 반영하기 위해 커밋하고 푸시한다.

```bash
# 변경 사항 커밋 및 푸시
git commit -m "chore: .vscode 폴더 Git 추적 제외 (캐시 삭제)"
git push
```

## 마무리

이렇게 커밋 후 푸시를 완료하면, 원격 저장소(GitHub 등)에서도 해당 폴더가 깔끔하게 삭제된다.
물론 로컬 PC에는 파일이 그대로 남아있으며, 앞으로는 `.gitignore`의 설정이 정상적으로 적용되어 해당 파일의 변경 사항이 더 이상 Git에 잡히지 않게 된다.

### 요약
- 이미 커밋된 파일은 `.gitignore`에 넣어도 계속 추적된다.
- `git rm -r --cached [폴더/파일명]` 명령어로 Git 캐시에서 삭제한다.
- 삭제된 상태를 커밋하고 푸시하면 완료된다.

### 다음 단계 (Next Step)
- `.gitignore` 파일에 어떤 내용들이 포함되어야 하는지 [gitignore.io](https://gitignore.io) 같은 도구를 활용해 프로젝트에 맞는 표준 설정을 구성해 볼 수 있다.
- 변경 사항이 제대로 무시되는지 `git status` 명령어를 통해 다시 한 번 확인한다.
- 팀원들에게 해당 내용을 공유하여 불필요한 설정 파일이 저장소에 올라가지 않도록 방지한다.
- 만약 단순 설정 파일이 아닌 API 키 등 민감한 정보가 올라가서 과거 커밋 히스토리까지 완전히 삭제해야 한다면 하단의 참고 문서를 확인한다.
