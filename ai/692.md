# Hermes Agent와 Discord 연동 가이드

최근 AI 에이전트를 외부 채팅 플랫폼과 연동하여 자동화된 워크플로우를 구축하려는 시도가 늘어나고 있다. 특히 NousResearch에서 공개한 Hermes Agent는 단순한 챗봇을 넘어 터미널 제어, 도구 사용, 메모리 관리 기능을 제공하므로 Discord와 연동할 경우 강력한 개인 비서로 활용이 가능하다.

- **대상 독자**: 터미널 환경에 익숙하며, Discord 봇을 생성하고 연동해 본 경험이 있는 개발자 또는 파워 유저
- **실행 환경**: macOS (Apple Silicon Mac mini 테스트 기준), Hermes Agent v1.x

이 글에서는 macOS(맥미니) 환경에서 Hermes Agent를 Discord 봇으로 구동하고 연결하는 필수 과정을 단계별로 알아본다.

---

## 1. 사전 준비: Hermes Agent 설치 (macOS)

Discord 연동을 시작하기 전에 맥미니 환경에 Hermes Agent를 설치해야 한다. 터미널에서 제공되는 공식 설치 스크립트를 활용한다.

```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

**참고 (설치 중 브라우저 팝업 창 무시):**
설치를 진행할 때 회원 가입을 유도하는 공식 페이지 브라우저 창이 열릴 수 있다. Discord 연동이 주 목적이라면 회원 가입 없이 창을 닫아도 설치와 구동에는 지장이 없다.

설치가 완료되면 쉘 설정을 다시 불러와 정상 작동 여부를 점검한다.

```bash
# bash를 사용하는 경우
source ~/.bashrc

# zsh를 사용하는 경우
source ~/.zshrc
```

## 2. LLM 모델 설정 및 추천 (Gemini API)

Hermes Agent가 답변을 생성하고 시스템 도구를 제어하기 위해서는 두뇌 역할을 할 LLM 설정이 선행되어야 한다. 터미널에서 `hermes model` 명령어를 입력하여 사용할 모델을 선택할 수 있다.

**모델 선택 실전 팁 (로컬 vs API)**
- **로컬 모델 (Ollama 등)**: Ollama를 이용해 Gemma4:e4b 오픈 가중치 모델을 로컬 환경에 띄워 연결했으나, Agent의 핵심인 **도구 호출(Tool Calling) 기능이 제대로 작동하지 않는 문제**가 발생했다.
- **API 모델 (Gemini 추천)**: 안정적인 도구 호출과 빠른 응답 속도를 원한다면 Gemini API 사용을 적극 권장한다. 특히 `gemini-3.1-flash-lite` 모델의 경우 하루 500회까지 무료 요청이 가능(2026년 6월 기준)하므로 비용 부담 없이 훌륭한 성능의 개인 비서를 구축할 수 있다.

## 3. Discord 앱 생성 및 봇 토큰 발급

Discord 측에서 Hermes Agent가 사용할 봇 계정을 생성하고 접속 권한을 얻는 과정이다.

1. **앱 생성**: 브라우저에서 [Discord Developer Portal](https://discord.com/developers/applications)에 로그인 후, 우측 상단의 **New Application**을 클릭하여 새로운 앱(예: `Hermes Agent`)을 생성한다. 생성 직후 화면에 나타나는 **Application ID**는 봇 초대에 사용되므로 별도로 기록해 둔다.
2. **봇 생성 및 토큰 발급**: 좌측 **Bot** 메뉴로 이동하여 봇을 활성화한다. **Public Bot**은 켜두고 **Require OAuth2 Code Grant**는 비활성화 상태를 유지한다. **Reset Token** 버튼을 눌러 발급된 **Bot Token**을 안전한 곳에 복사해 둔다.

**주의**: Bot Token은 절대 외부에 노출되어서는 안 되며, 유출이 의심될 경우 즉시 재발급해야 한다.

## 4. Privileged Gateway Intents 활성화

Hermes Agent가 사용자의 메시지를 정상적으로 수신하고 반응하기 위해서는 권한 인텐트(Intent) 설정이 필수적이다.

1. **Bot** 메뉴 하단의 **Privileged Gateway Intents** 섹션으로 이동한다.
2. 역할 기반 제어를 위해 **Server Members Intent**를 활성화한다.
3. 봇이 채팅 텍스트를 읽을 수 있도록 **Message Content Intent**를 반드시 활성화한다.
4. 하단의 **Save Changes**를 눌러 설정을 저장한다.

이 중 **Message Content Intent**가 꺼져 있으면 봇이 온라인 상태임에도 사용자의 메시지에 아무런 응답을 하지 못하는 문제가 발생하므로 각별히 주의한다.

## 5. Discord 서버에 봇 초대

생성한 봇을 자신이 관리하는 Discord 서버로 초대하여 대화 환경을 구성한다. 아래 URL의 `YOUR_APP_ID` 부분을 앞서 기록해 둔 **Application ID**로 치환하여 브라우저에 입력한다.

```text
https://discord.com/oauth2/authorize?client_id=YOUR_APP_ID&scope=bot+applications.commands&permissions=274878286912
```

이 권한 번호(`274878286912`)는 메시지 송수신, 파일 첨부, 스레드 이용 등 Hermes Agent 구동에 필요한 기본적인 제어 권한을 모두 포함하고 있다.

## 6. 보안을 위한 Discord User ID 설정

Hermes Agent는 강력한 시스템 제어 권한을 가질 수 있으므로, 인가된 사용자만 봇을 다룰 수 있도록 자신의 User ID를 허용 목록에 등록해야 한다.

1. **개발자 모드 활성화**: Discord 앱의 **사용자 설정**에서 **고급** 메뉴로 들어가 **개발자 모드**를 활성화한다.
2. **User ID 복사**: 본인의 프로필을 우클릭하여 **사용자 ID 복사**를 선택한 후 메모해 둔다.

**참고 (ID 복사가 안 되고 사용자 이름만 복사될 때 꿀팁):**
디스코드 UI 업데이트 등의 이유로 숫자 ID 복사가 불가능하다면, 우선 정확한 ID 등록을 보류하고 봇 실행 단계(7단계)를 먼저 진행한다. 이후 봇에게 메시지를 보내면 최초 대화는 이뤄지지만 후속 대화가 단절될 수 있다. 이때 맥미니 터미널에서 로그(`tail -f ~/.hermes/logs/gateway.log`)를 확인하면 본인 이름과 매핑된 실제 숫자 형태의 User ID를 발견할 수 있다. 이 숫자를 `.env` 설정에 입력하고 재구동하면 대화를 정상적으로 이어갈 수 있다.

## 7. Hermes Gateway 환경 변수 등록 및 실행

발급받은 Bot Token과 확인한 User ID를 Hermes 설정 파일에 등록하고 게이트웨이를 실행한다.

**환경 변수 설정**
맥미니 터미널에서 `~/.hermes/.env` 파일을 편집기로 열어 정보를 기입한다. 복수의 사용자를 허용하려면 쉼표로 구분한다. 시스템 보호를 위해 모든 사용자를 허용하는 `GATEWAY_ALLOW_ALL_USERS=true` 설정은 권장하지 않는다.

```ini
# ~/.hermes/.env 파일 내용

DISCORD_BOT_TOKEN=발급받은_디스코드_봇_토큰
DISCORD_ALLOWED_USERS=복사해둔_본인의_User_ID
```

**Gateway 구동 및 서비스 등록**
설정이 완료되었다면 터미널에서 구동 테스트를 진행한다. 디스코드 서버에서 봇이 온라인으로 전환되고 멘션에 응답하는지 점검한다.

```bash
hermes gateway
```

성공적으로 작동한다면, 맥미니에서 터미널을 종료해도 항상 봇이 실행되도록 macOS의 `launchd` 서비스로 등록하는 것을 권장한다.

```bash
hermes gateway install
hermes gateway start
hermes gateway status
```

동작 중 에러나 기록을 확인하려면 `tail -f ~/.hermes/logs/gateway.log` 명령어를 사용한다.

## 마무리 및 요약

- **보안 설정 우선**: Discord Developer Portal에서 봇 생성 시 토큰 관리에 유의하고, 반드시 Message Content Intent를 활성화해야 한다.
- **사용자 제어 보장**: 봇의 무분별한 사용을 막기 위해 본인의 Discord User ID를 `.env` 파일에 정확히 등록하며, 로그를 활용한 우회 확인 방법을 적극 활용한다.
- **백그라운드 운용**: 테스트 완료 후 macOS 서비스에 등록하여 봇이 항상 대기할 수 있는 안정적인 환경을 구축한다.

Hermes Agent의 Discord 연동은 개발자에게 매끄럽고 강력한 봇 인터페이스를 제공한다. 위 과정을 통해 연동을 성공적으로 마쳤다면, 제공되는 다양한 도구와 기능을 응용하여 자신만의 자동화 환경을 구축해 보길 바란다. 적용 과정에서 오류가 발생하거나 궁금한 점이 있다면 언제든 댓글을 남겨주길 바란다!
